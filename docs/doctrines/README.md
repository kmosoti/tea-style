# Doctrines

## Purpose
Hold cross-domain TEA doctrine that applies across coding, automation, agent architecture, and observability.

## What belongs here
- repository-wide engineering doctrine
- cross-domain mental models
- canonical terminology that several domains should share
- decision-style guidance that is broader than one technical area

## What does not belong here
- domain-specific standards that belong under `domains/<domain>/`
- long notes dumps
- implementation code
- vendor-specific current-state summaries
